# Исследование сортировок строк

В работе сравнивались обычные сортировки строк и алгоритмы, которые учитывают посимвольное устройство строк:

- QuickSort;
- MergeSort;
- тернарный String QuickSort;
- String MergeSort с LCP;
- MSD Radix Sort;
- MSD Radix Sort с переключением на String QuickSort.

Для тестов использовались случайные, обратно отсортированные и почти отсортированные массивы. Также был добавлен отдельный набор строк с общими префиксами, чтобы лучше увидеть разницу между обычными и строковыми сортировками.

Размеры массивов брались от 100 до 3000 строк с шагом 100. Для каждого запуска измерялось среднее время работы и количество обращений к символам строк.

## Выводы

На случайных строках быстрее всего в большинстве случаев работал тернарный String QuickSort. Он оказался удачным вариантом именно для строк, потому что не сравнивает одни и те же префиксы слишком много раз.

MSD Radix Sort делал мало обращений к символам, особенно на случайных строках. При этом по времени он не всегда был самым быстрым, потому что у него есть дополнительные расходы на массивы счетчиков и распределение строк.

Версия MSD Radix Sort с переключением на String QuickSort обычно работала быстрее обычной версии MSD на небольших подмассивах. Это хорошо видно по графикам времени.

Обычный QuickSort на этих данных показал себя достаточно неплохо, потому что опорный элемент выбирался из середины. Поэтому на обратно отсортированных массивах он не ушел в плохой случай.

На строках с общими префиксами преимущество адаптированных алгоритмов заметнее всего. Обычные сортировки много раз заново сравнивают одинаковые начала строк, а String QuickSort, LCP MergeSort и MSD Radix Sort используют эту информацию эффективнее.

String MergeSort с LCP уменьшил количество посимвольных сравнений, но по времени получился не самым быстрым. Скорее всего, это связано с тем, что реализация достаточно простая и в ней есть лишние копирования.

В целом по результатам видно, что специализированные алгоритмы сортировки строк действительно уменьшают число операций над символами. По времени лучший результат чаще всего дал тернарный String QuickSort.
